# Uber网约车平台运营数据分析项目

## 项目背景

本项目基于 Uber Trips Dataset（50,000条订单数据），利用 SQL Server 对网约车平台运营情况进行分析，从订单规模、城市运营表现、用户行为及平台运营效率等多个维度挖掘业务价值，并提出优化建议。

---

## 技术栈

* SQL Server
* SQL
* Excel

---

## 数据说明

数据量：

* 50,000条订单记录

主要字段：

| 字段             | 含义   |
| -------------- | ---- |
| trip_id        | 订单ID |
| rider_id       | 用户ID |
| city           | 城市   |
| distance_km    | 行程距离 |
| fare_amount    | 订单金额 |
| status         | 订单状态 |
| payment_method | 支付方式 |
| pickup_time    | 下单时间 |

## 核心分析结果

### 平台整体情况

| 指标      | 结果      |
| ------- | ------- |
| 总订单量    | 50,000  |
| 总GMV    | 798,810 |
| 平均客单价   | 15.98   |
| 平均订单距离  | 7.01km  |
| 平均每公里收入 | 2.61    |

### 订单状态分析

| 状态        | 数量     |
| --------- | ------ |
| Completed | 42,540 |
| Cancelled | 4,984  |
| No-Show   | 2,476  |

平台整体取消率约为14.92%。

### 城市运营分析

* Boston订单量最高（8,454）
* Boston GMV最高（135,704）
* Los Angeles取消率最高（15.42%）
* Seattle客单价最高（16.06）

### 用户价值分析

高价值用户消费金额约80~120元，订单数约4~6单，具备进一步会员运营和精准营销价值。

---

## 业务结论

1. 平台整体运营稳定，城市间差异较小。
2. 平台取消率约15%，仍有优化空间。
3. 用户以中短途出行为主，平均订单距离约7公里。
4. 支付方式分布均衡，用户支付习惯多样化。
5. 可针对高价值用户建立会员体系，提高用户留存率。
